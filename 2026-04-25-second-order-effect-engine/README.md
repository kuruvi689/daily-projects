# Second-Order Effect Engine (SOEE)\n\n## Project Description\n\nThe **Second-Order Effect Engine (SOEE)** is a modular Python framework designed to simulate and analyze the cascading consequences of initial actions or decisions within complex interconnected systems. Operating as a logic-based system, SOEE helps strategic planners, decision-makers, and systems architects anticipate not just immediate impacts, but also the subsequent, indirect effects that can arise across various domains.\n\nBy modeling relationships between different "factors" (e.g., business metrics, psychological states, market conditions), SOEE provides a structured approach to understand how a single change can ripple through an entire system, revealing potential long-term benefits, risks, and unintended consequences. This tool enhances "Strategic Sharpness" by sharpening the "mind" to identify high-leverage opportunities and mitigate unseen risks before they materialize.\n\n## Features\n\n-   **Graph-based Factor Modeling**: Define a network of interconnected factors with specified positive or negative relationships and their respective weights (strength of impact).\n-   **Action Definition**: Model initial actions or decisions by specifying their direct impacts on a subset of factors.\n-   **Cascading Effect Simulation**: Simulate the propagation of impacts through the defined factor graph, tracking effects across multiple orders (direct, indirect, and subsequent).\n-   **Impact Analysis**: Quantify the total cumulative impact on each factor and detail the order in which these impacts occur.\n-   **Modular Design**: Core components (`EffectGraph`, `Action`, `SecondOrderEffectSimulator`) are designed for reusability and integration into larger systems.\n-   **SaaS-Ready Potential**: Can be deployed as a micro-service, allowing other applications to query it with different scenarios and receive structured impact analysis.\n\n## SaaS Potential\n\nThe SOEE has significant potential as a micro-service. Organizations could integrate it into their strategic planning tools, risk management platforms, or policy analysis systems. A paid API could allow users to:\n-   Upload and manage custom effect graphs for different business units or industries.\n-   Submit proposed actions and receive real-time, quantified second-order effect analyses.\n-   Compare the long-term impact of different strategic choices.\nThis service could provide invaluable foresight, enabling proactive decision-making and reducing unforeseen negative outcomes.\n\n## Installation\n\nThis project uses standard Python libraries. No special installation is required beyond a Python 3 environment.\n\n```bash\ngit clone https://github.com/yourusername/second-order-effect-engine.git\ncd second-order-effect-engine\n```\n\n## Usage\n\n### 1. Define Your Effect Graph\n\nCreate an instance of `EffectGraph` and add factors (nodes) and directed relationships (edges) between them. Relationships require an `impact_type` ("positive" or "negative") and a `weight` (0.0 to 1.0) indicating the strength.\n\n```python\nfrom main import EffectGraph\n\neffect_graph = EffectGraph()\neffect_graph.add_factor("Customer Satisfaction")\neffect_graph.add_factor("Product Quality")\neffect_graph.add_relationship("Product Quality", "Customer Satisfaction", "positive", 0.8)\n# ... add more factors and relationships\n```\n\nYou can also load graphs from JSON:\n```python\njson_config = """\n{\n  \"Customer Satisfaction\": {\n    \"Brand Reputation\": {\"type\": \"positive\", \"weight\": 0.7}\n  },\n  \"Brand Reputation\": {}\n}\n"""\neffect_graph = EffectGraph.from_json(json_config)\n```\n\n### 2. Define an Action\n\nCreate an instance of `Action` with a name and a dictionary of direct impacts. Each direct impact is a factor name mapped to a float value representing its initial change (e.g., `0.5` for a positive change, `-0.3` for a negative change).\n\n```python\nfrom main import Action\n\naction_cost_cut = Action(\n    name="Implement Cost-Cutting Measure",\n    direct_impacts={\n        "Operational Costs": -0.4,\n        "Employee Morale": -0.3\n    }\n)\n```\n\n### 3. Run the Simulation\n\nInitialize the `SecondOrderEffectSimulator` with your `EffectGraph` and call the `simulate` method with your `Action` and an optional `max_depth`.\n\n```python\nfrom main import SecondOrderEffectSimulator\n\nsimulator = SecondOrderEffectSimulator(effect_graph)\nresults = simulator.simulate(action_cost_cut, max_depth=3)\n\n# Print the results\nfor factor, data in results.items():\n    print(f"\nFactor: {factor}")\n    print(f"  Total Impact: {data['total_impact']:.2f}")\n    for detail in data['impact_details']:\n        print(f"    Order {detail['order']} Impact: {detail['value']:.2f}")\n```\n\n## Example Walkthrough\n\nSee the `if __name__ == "__main__":` block in `main.py` for a complete example demonstrating a cost-cutting action and an R&D investment action, and their simulated effects.\n\n## Contributing\n\nContributions are welcome!
