# FinRAG Assistant: AI-Powered Financial Document Analysis

## Project Summary

The **FinRAG Assistant** is a strategic Python tool designed to leverage Retrieval-Augmented Generation (RAG) for in-depth analysis of financial documents, such as earnings reports, annual filings, or investor presentations. It enables users to ask specific questions about these documents and receive concise, contextually relevant answers generated by an LLM, reducing manual research time and enhancing insight extraction.

## Features

*   **PDF Document Ingestion:** Loads and processes financial documents in PDF format.
*   **Intelligent Text Chunking:** Breaks down large documents into manageable, context-rich segments.
*   **Vector Store Creation:** Generates embeddings for document chunks and stores them in a FAISS vector database for efficient retrieval.
*   **Retrieval-Augmented Generation (RAG):** Combines contextually relevant document snippets with a Large Language Model (LLM) to generate accurate and concise answers to user queries.
*   **Financial Query Specialization:** Optimized with a prompt tailored for financial analysis, ensuring relevant and factual responses.
*   **Modular Design:** Separates document processing, embedding, vector store management, and LLM interaction into distinct components for reusability and scalability.

## Installation and Setup

1.  **Clone the repository (or save the script):**
    bash
    git clone <repository-url> # If part of a larger repo
    cd <project-directory>
    
    *(If saving just the script, ensure `finrag_assistant.py` is in your working directory.)*

2.  **Install dependencies:**
    bash
    pip install -r requirements.txt
    

3.  **Set up your OpenAI API Key:**
    The project requires an OpenAI API key for embeddings and LLM interaction. Set it as an environment variable (replace `your_openai_api_key_here` with your actual key):
    bash
    export OPENAI_API_KEY="your_openai_api_key_here"
    # For Windows Command Prompt: set OPENAI_API_KEY="your_openai_api_key_here"
    # For Windows PowerShell: $env:OPENAI_API_KEY="your_openai_api_key_here"
    
    *Note: Using environment variables is crucial for security and best practice.*

4.  **Prepare your Financial Document:**
    Place a PDF document (e.g., an earnings report like `2023_company_earnings.pdf`) in the same directory as the `finrag_assistant.py` script, or update the `DOCUMENT_PATH` variable in the script to point to your file's absolute path. For demonstration, name it `sample_earnings_report.pdf`.

## Usage

1.  **Run the script:**
    bash
    python finrag_assistant.py
    

2.  **Interact:**
    The script will load your specified PDF, process it, build the RAG system, and then execute a set of predefined example queries. You can modify the `queries` list in the `if __name__ == "__main__":` block to test your own questions.

    Example output might look like:
    
    Loading document from sample_earnings_report.pdf...
    Loaded 10 pages.
    Splitting document into chunks (size=1000, overlap=200)...
    Created 25 chunks.
    Building vector store...
    Vector store built. Setting up RAG chain...
    RAG chain setup complete.

    Querying: What was the net income for Q4 2023?
    Answer: The net income for Q4 2023 was $X million, as stated on page Y of the report.

    Querying: Can you summarize the company's performance in the last fiscal year?
    Answer: In the last fiscal year, the company reported strong revenue growth of A%, with net income increasing by B%. Key drivers included...
    

## Code Structure

*   `DocumentProcessor` class: Handles the loading and chunking of PDF documents.
*   `FinRAGAssistant` class: Manages the creation of embeddings, the FAISS vector store, the OpenAI LLM integration, and orchestrates the RAG query process.
*   `if __name__ == "__main__":` block: Contains the main execution logic, configuration, and example usage.
